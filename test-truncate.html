What's happening?

  • Case 1: Default min-width: auto means "my minimum width = content width"
  • The yellow div expands to fit the full text width
  • Since the container never constrains the text, truncate never triggers
  • Case 2: min-w-0 allows the div to shrink smaller than content
  • The text overflows the div boundary, triggering text-overflow: ellipsis

Technical breakdown

truncate in Tailwind equals:

overflow: hidden;
text-overflow: ellipsis;
white-space: nowrap;

For ellipsis to work, you need:

  1. A width constraint (explicit or from parent)
  2. Content that overflows that constraint
  3. overflow: hidden

Flexbox default behavior prevents this:

min-width: auto ‚Üí element can't shrink below content width ‚Üí no overflow ‚Üí no ellipsis

Solution:

min-w-0 ‚Üí element can shrink to 0 ‚Üí overflow happens ‚Üí ellipsis works
